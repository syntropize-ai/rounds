# Feed

Feed is the operator inbox for events Rounds thinks you may need to review.

## What appears in Feed

- completed investigations;
- detected anomalies;
- change-impact items;
- future event types produced by connectors or background agents.

Each item has a severity, read state, timestamp, and link to the related resource when available.

## Use Feed

Open **Feed** from the sidebar when it is enabled in your app build, or follow feed links from related views.

You can:

- filter by severity;
- filter by unread or read state;
- mark one item read;
- mark all visible unread items read;
- refresh the list;
- open the related investigation or follow-up action.

## When to use it

Use Feed for asynchronous awareness: things Rounds found while you were not actively chatting, such as completed reports or notable system changes.

Use **Action Center** instead when the item needs approval before work can continue.
